# Bridge CRM — Project Specs

---

## 1 — Product Vision

**Bridge CRM is the CRM for people who hate CRMs.**

HubSpot and Salesforce are too big, too complex, and too expensive for SMBs. Most small businesses track their customers in Excel spreadsheets — until the file crashes and they lose everything. Bridge CRM is for them.

**Core philosophy:**
- **As easy as Excel to set up.** A business owner with zero technical knowledge should be able to sign up, import their Excel data, and be productive within 15 minutes.
- **So simple a 13-year-old can use it.** No jargon, no hidden menus, no 50-page manuals. Every screen tells you exactly what to do next.
- **The CRM works for you, not the other way around.** Bridge doesn't just store data — it actively guides users. It tells you who to call, what follow-up is overdue, and what deal needs attention. A central "work queue" drives daily workflow.
- **Grow step by step.** Start with the basics (contacts + pipeline). Add more when you're ready (email sync, documents, invoicing). Never overwhelmed.
- **Data you can trust.** Clean, consistent, deduplicated data. When you import from Excel, HubSpot, or Salesforce, Bridge resolves conflicts transparently and tells you what it did.

**Target user:** SMB business owners and their small teams (1–25 people) who:
- Currently use Excel/Google Sheets to track customers
- Find HubSpot/Salesforce too complex and expensive
- Don't have a dedicated IT person or CRM admin
- Want something that "just works"

---

## 2 — Product Principles

1. **Less is more.** Every feature must earn its place. Fewer options, clearer defaults. If a user has to think about a setting, we chose wrong.
2. **Always show the next step.** Every screen should answer: "What should I do now?" — via a work queue, smart suggestions, or a clear call-to-action.
3. **Opinionated defaults.** Ship with sensible pipeline stages, field names, and templates. Users can customize later, but they should never have to before they start.
4. **Progressive disclosure.** Show the basics first. Advanced features reveal themselves as the user grows. Never front-load complexity.
5. **Data quality over data quantity.** Duplicate detection, merge suggestions, clean imports. Trustworthy data beats more data.

## 4 — Growth Phases

Bridge CRM grows with the business. Each phase builds on the previous one.

### Phase 1: Sales Foundation (MVP)
The basics that replace Excel. This is what we build first.

- **Contacts & Companies** — store customer data (name, email, phone, company, notes)
- **Pipeline / Kanban** — visual board showing where every deal stands. Drag cards between stages freely.
- **Work Queue** — a daily to-do list generated by the CRM. "Call Sophie back (3 days overdue)." "Send proposal to DataCore." "Follow up on Fintech Group deal."
- **Activity timeline** — every interaction with a customer in one chronological view (calls, emails, notes, deal changes)
- **CSV import** — upload your Excel file, map the columns, done. Smart duplicate detection on import.
- **Search** — one search bar that finds anything: contacts, companies, deals, notes
- **Dashboard** — simple overview: open deals, overdue tasks, recent activity, pipeline value
- **Onboarding wizard** — 3 steps: create account, name your company, import your data (or start empty with sample data)

### Phase 2: Communication
Connect email and calendar so everything lives in one place.

- **Gmail sync** — sent/received emails auto-log as CRM activities on the contact timeline
- **Google Calendar two-way sync** — CRM activities ↔ Google Calendar events, follow-up tasks become calendar reminders
- **Email from CRM** — send a quick email without leaving the contact page
- **Business card scanner** — snap a photo (jpg, heic, pdf), OCR extracts the data, match or create contact

### Phase 3: Documents & Implementation
Track the full customer lifecycle — not just the sale, but what comes after.

- **File attachments** — upload quotes, contracts, signed documents, installation photos to any contact or deal
- **Document stages** — track which documents are pending, signed, or expired
- **Implementation tracking** — after a deal is won, track the onboarding: installation data, delivery status, setup photos
- **Notes & checklists** — add structured checklists to deals ("Site survey done", "Contract signed", "Equipment installed")

### Phase 4: Integrations & Sync
Bridge to other systems.

- **HubSpot import/export** — CSV first, then API sync
- **Salesforce import/export** — CSV first, then API sync
- **ERP connector** — map fields, schedule sync, track sync health
- **Bi-directional sync** — conflict resolution with clear source hierarchy and user notification

### Phase 5: Marketing & Automation
Help small teams do marketing without a marketing team.

- **Email templates** — create reusable templates for common emails
- **Email sequences** — automated follow-up sequences (3 emails over 2 weeks)
- **Simple segments** — filter contacts by stage, tag, or custom field
- **Campaign tracking** — open/click rates on sent emails

### Phase 6: Advanced / Enterprise
For growing teams that need more.

- **Web scraper** — user-configurable profiles to find leads from websites
- **Workflow automation** — "When a deal moves to 'Won', create an implementation checklist"
- **Advanced reporting** — custom reports, export to PDF
- **Invoicing link** — connect to invoicing/accounting tools
- **Inventory connection** — link deals to stock/product availability
- **SSO, audit logs, advanced permissions**

## 7 — Work Queue (key differentiator)

The work queue is what makes Bridge different from a spreadsheet. It's a smart to-do list that tells each user what to do today.

**How it works:**
1. The queue pulls from `activities` where `is_task = true AND completed = false`
2. It also auto-generates items based on rules:
   - Contacts with `next_follow_up` in the past -> "Follow up with [name]"
   - Deals that haven't been updated in 7+ days -> "Check in on [deal title]"
   - New leads from imports that haven't been contacted -> "Review new lead: [name]"
3. Items are sorted by urgency: overdue first, then due today, then upcoming
4. Users can snooze, complete, or dismiss items
5. Completing a task automatically logs it as an activity on the contact/deal timeline

**Where it shows up:**
- Dashboard: top section, "Your tasks for today"
- Sidebar: small badge with count of overdue items
- Contact/deal detail: relevant tasks inline

## 10 — UI Design Principles

- **Dark theme** — premium, modern feel. Easy on the eyes for all-day use.
- **Minimal chrome** — no unnecessary borders, shadows, or decoration. Let the data breathe.
- **Consistent spacing** — use Tailwind's spacing scale (4, 8, 12, 16, 24, 32, 48). No random pixel values.
- **Clear hierarchy** — headings, subheadings, body text. Use font weight and size, not color, for emphasis.
- **One primary action per screen** — every page has one obvious thing to do. The button for it is prominent.
- **Inline editing where possible** — don't make users open a modal to change a phone number. Click, edit, save.
- **Instant feedback** — optimistic updates for drag/drop and inline edits. Show toast for saves.
- **Empty states matter** — when there's no data, show a friendly message and a clear action: "No contacts yet. Import your Excel file or add your first contact."
- **No jargon** — "Customers" not "Entities". "Pipeline" not "Opportunity Funnel". "Tasks" not "Work Items".

**Step 1: Sign up**

After onboarding, land on the dashboard with the work queue front and center.

## 11b — Role-Based Dashboards

Bridge CRM shows a different dashboard depending on your role. Both views use the same underlying data (contacts, companies, deals, activities) but present it differently.

### Account Manager Dashboard (`/dashboard/account-manager`)

The AM view is personal — focused on **your** accounts, **your** tasks, **your** pipeline.

**KPI cards:**
- Total ARR managed (sum of deal values for your accounts)
- At-Risk ARR (accounts with no activity in 30+ days)
- Renewals this month (deals with expected_close_date in current month)
- Open tasks (incomplete tasks assigned to you)

**Account Health Table:**
Each company you own, scored green/amber/red based on last contact date:
- Green: contacted within 14 days
- Amber: 14–30 days since last contact
- Red: 30+ days (at risk)
Columns: Account name, Segment (category), ARR (deal value sum), Last contact, Health, Trend (up/stable/down based on deal changes)

**Today's Tasks:**
Same as work queue but styled by urgency — overdue (red), due today (amber), upcoming (grey).

**My Pipeline by Stage:**
Only your deals, grouped by kanban stage with count + total value per stage.

**Upcoming Renewals:**
Deals with expected_close_date in the next 90 days, sorted by date, colour-coded:
- Red: under 30 days
- Amber: 30–60 days
- Green: 60+ days

### Sales Manager Dashboard (`/dashboard/sales-manager`)

The SM view is about the **team** — performance, coaching, forecasting.

**KPI cards:**
- Team quota attainment % (closed won vs team quota target)
- Total closed won (sum of deals in "Won" stages)
- Pipeline coverage (pipeline value / remaining quota)
- Deals at risk (high-value deals with no activity in 7+ days)

**Rep Performance Table:**
Each user in the tenant, showing:
- Attainment % (closed won / quota)
- Closed revenue
- Pipeline value (open deals)
- Activity score (activity count relative to team average)
- Avg deal size

**Deals at Risk:**
Deals with no update in 7+ days that are not won/lost, showing company, rep, stage, value, days stalled.

**Pipeline Funnel:**
All deals across the team, grouped by stage with count, value, and drop-off % from previous stage.

**Q2 Forecast Table:**
- Quota (team target, stored in tenant settings)
- Closed won to date
- Pipeline commit (deals in Negotiation+)
- Best case (all open pipeline)
- Gap to quota

### Database additions for role dashboards

New columns:
- `users.quota` (numeric) — quarterly revenue target per user
- `tenants.settings.team_quota` (jsonb) — team-level quarterly target

No new tables needed — everything derives from existing deals, activities, contacts, and companies.

## 16 — Google Integration Spec (Phase 2a)

### Overview

Integrate Gmail and Google Calendar into Bridge CRM so account managers can:
1. See sent/received emails auto-logged as activities on contact timelines
2. Sync CRM activities ↔ Google Calendar events (two-way)
3. Create follow-up tasks in the CRM that automatically appear as calendar reminders

### Architecture

**OAuth 2.0 flow:**
- User clicks "Connect Gmail & Calendar" in Settings → Integrations
- Redirect to Google consent screen requesting `gmail.readonly`, `gmail.send`, `calendar.events` scopes
- Callback stores encrypted tokens per user in `user_integrations` table
- Tokens auto-refresh via refresh_token when access_token expires

**Data model — new tables:**

```
user_integrations {
  id: uuid (PK)
  user_id: uuid (FK -> profiles)
  tenant_id: uuid (FK -> tenants)
  provider: text ('google')
  access_token: text (encrypted)
  refresh_token: text (encrypted)
  token_expires_at: timestamptz
  scopes: text[]
  email_address: text        -- the connected Google account email
  sync_enabled: boolean (default true)
  last_sync_at: timestamptz
  created_at: timestamptz
  updated_at: timestamptz
}

integration_sync_log {
  id: uuid (PK)
  user_integration_id: uuid (FK -> user_integrations)
  sync_type: text ('gmail_incoming', 'gmail_sent', 'calendar_push', 'calendar_pull')
  external_id: text          -- Gmail message ID or Calendar event ID
  activity_id: uuid (FK -> activities, nullable)
  synced_at: timestamptz
}
```

RLS: both tables scoped to `tenant_id = get_user_tenant_id()`.

### Gmail Sync

**How it works:**
1. On connect, do an initial sync of sent emails from the last 30 days
2. Periodic sync (triggered on page load or manual refresh) fetches new messages since `last_sync_at`
3. For each email, match recipient/sender to a CRM contact by email address
4. If matched, create an activity: type='email', title='Email: [subject]', description=snippet, contact_id=matched contact, company_id=contact's company
5. Deduplication: check `integration_sync_log.external_id` before creating activity

**API routes:**
- `GET /api/integrations/google/auth` — generates Google OAuth URL, redirects user
- `GET /api/integrations/google/callback` — exchanges code for tokens, stores in DB
- `POST /api/integrations/gmail/sync` — triggers sync for current user
- `DELETE /api/integrations/google/disconnect` — revokes tokens, deletes integration record

### Google Calendar Sync

**Two-way sync:**

*CRM → Calendar (push):*
- When a user creates an activity with `is_task=true` and `due_date` set, create a Google Calendar event
- Event title = activity title, time = due_date, description = activity description
- Store the calendar event ID in `integration_sync_log` for future updates/deletes

*Calendar → CRM (pull):*
- On sync, fetch events from Google Calendar since last sync
- Match attendees to CRM contacts by email
- Create activity: type='meeting', title=event summary, contact_id=matched contact
- Skip events already in sync_log (deduplication)

**Follow-up → Reminder flow:**
- When a user marks an activity as a follow-up task (is_task=true, due_date set), and they have Google Calendar connected
- Automatically create a calendar event with a 15-minute reminder
- If the task is completed in CRM, mark the calendar event as completed/cancelled

### Settings UI

New tab in Settings page: **Integrations**

Shows integration cards:
- **Gmail** — Connect/Disconnect button, last sync time, sync status indicator

```
GOOGLE_CLIENT_ID=           # From Google Cloud Console
GOOGLE_CLIENT_SECRET=       # From Google Cloud Console
GOOGLE_REDIRECT_URI=        # e.g. http://localhost:3000/api/integrations/google/callback
```

### npm package

`googleapis` — official Google API client for Node.js

### File structure

```
/lib/integrations/google/
  auth.ts          — OAuth helpers (getAuthUrl, exchangeCode, refreshToken)
  gmail.ts         — Gmail sync logic (fetchEmails, matchToContact, createActivity)
  calendar.ts      — Calendar sync logic (pushEvent, pullEvents, deleteEvent)

/app/api/integrations/google/
  auth/route.ts           — GET: redirect to Google consent
  callback/route.ts       — GET: handle OAuth callback
  disconnect/route.ts     — DELETE: revoke + delete

/app/api/integrations/gmail/
  sync/route.ts           — POST: trigger Gmail sync

/app/api/integrations/calendar/
  sync/route.ts           — POST: trigger Calendar sync
  webhook/route.ts        — POST: receive Google Calendar push notifications (future)

/app/(dashboard)/settings/integrations/
  page.tsx                — Integration cards UI
```
